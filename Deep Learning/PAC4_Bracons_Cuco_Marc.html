Informaci de la prctica

Les xarxes generatives adversries (GANs) ens permeten generar dades como imatges, vdeos o udio a partir d'una entrada aleatria. Normalment, aquesta entrada es mostreja d'una distribuci normal abans de passar per una srie de transformacions que la converteixen en quelcom plausible, com una imatge, un so o un clip de vdeo.

Tot i aix, una DCGAN senzilla no ens permet controlar l'aparena (per exemple, la classe) de les mostres generades. Per exemple, si una GAN genera dgits manuscrits, no podrem triar quin nmero volem que aparegui: els resultats serien aleatoris.

Per poder controlar all que es genera, necessitem condicionar la sortida de la GAN a una entrada semntica, com la classe d'una imatge.

En aquesta prctica (PAC), construirem una GAN Condicional que sigui capa de generar dgits manuscrits condicionats a una classe concreta.

  • Utilitza Sequential() de keras per definir ambds models pas a pas, de forma clara i ordenada.
  • Discriminador (aproximadament 56 capes):
    • L'entrada ha de tenir forma (28, 28, discriminator_in_channels), s a dir, la imatge original a escala de grisos ms la informaci de classe afegida com a canals.

    • Al final, utilitza una capa de pooling global per reduir el mapa a un vector i una capa Dense que torni un nic valor (real o fals).

    • L'entrada ha de tenir forma (generator_in_channels,), s a dir, el vector latent concatenat amb la classe codificada.
    • Comena amb una capa Dense que converteixi aquest vector en un volum inicial (per exemple, 77canals).
    • Utilitza Reshape per transformar el vector pla en un mapa 2D amb profunditat.
    • Aplica diverses capes Conv2DTranspose (transposades) per anar ampliant la mida de la imatge de forma progressiva.
    • Desprs de cada transposici, utilitza BatchNormalization i LeakyReLU per millorar l'estabilitat i la qualitat de la imatge.

5. Implementació de la classe ConditionalGAN¶

En aquesta secci es defineix una classe personalitzada que estn keras.Model, i permet controlar manualment el procs d'entrenament d'una GAN condicional.

A diferncia dels models estndard, una GAN requereix dues xarxes entrenades de forma alternada:

  • El discriminador, que intenta distingir entre imatges reals i generades.
  • El generador, que intenta crear imatges prou realistes per enganyar el discriminador.

El mtode train_step est estructurat per blocs numerats (apartats) per entendre cada etapa de l'entrenament, des de la preparaci de dades fins a l'actualitzaci de pesos.

Una GAN condicional hi afegeix una capa de complexitat: s'entrena amb imatges i tamb amb etiquetes que indiquen la classe desitjada.

Llegeix atentament el codi i respon les preguntes segents:

class ConditionalGAN(keras.Model):
    def __init__(self, discriminator, generator, latent_dim):
        super().__init__()
        # Assignar discriminador i generador
        self.discriminator = discriminator
        self.generator = generator

        # Dimensi del vector d'entrada aleatori (soroll)
        self.latent_dim = latent_dim

        # Generador de llavors per a la reproducibilitat
        self.seed_generator = keras.random.SeedGenerator(1337)

        # Mtriques per al seguiment de les prdues del generador i del discriminador
        self.gen_loss_tracker = keras.metrics.Mean(name="generator_loss")
        self.disc_loss_tracker = keras.metrics.Mean(name="discriminator_loss")

    @property
    def metrics(self):
        # Llista de mtricas que es reportaran durant l'entrenament
        return [self.gen_loss_tracker, self.disc_loss_tracker]

    def compile(self, d_optimizer, g_optimizer, loss_fn):
        # Configurar optimizadors i funci de prdua
        super().compile()
        self.d_optimizer = d_optimizer
        self.g_optimizer = g_optimizer
        self.loss_fn = loss_fn

    def train_step(self, data):
        # Desempaquetar les dades reals (imatges i etiquetes one-hot)
        real_images, one_hot_labels = data

        # apartat 1: Expandir i repetir etiquetes
        image_one_hot_labels = one_hot_labels[:, :, None, None]
        image_one_hot_labels = ops.repeat(
            image_one_hot_labels, repeats=[image_size * image_size]
        )
        image_one_hot_labels = ops.reshape(
            image_one_hot_labels, (-1, image_size, image_size, num_classes)
        )

        # apartat 2: Generar vectors latents aleatoris i concatenar-los amb etiquetes
        batch_size = ops.shape(real_images)[0]
        random_latent_vectors = keras.random.normal(
            shape=(batch_size, self.latent_dim), seed=self.seed_generator
        )
        random_vector_labels = ops.concatenate(
            [random_latent_vectors, one_hot_labels], axis=1
        )

        # apartat 3: Generar imatges falses a partir del generador
        generated_images = self.generator(random_vector_labels)

        # apartat 4: Concatenar imatges falses amb les seves etiquetes
        fake_image_and_labels = ops.concatenate(
            [generated_images, image_one_hot_labels], -1
        )

        # apartat 5: Concatenar imatges reals amb les seves etiquetes
        real_image_and_labels = ops.concatenate([real_images, image_one_hot_labels], -1)

        # apartat 6: Unir el batch d'imatges reals i falses
        combined_images = ops.concatenate(
            [fake_image_and_labels, real_image_and_labels], axis=0
        )

        # apartat 7: Crear etiquetes: 1 per a falses, 0 per a reals
        labels = ops.concatenate(
            [ops.ones((batch_size, 1)), ops.zeros((batch_size, 1))], axis=0
        )

        # apartat 8: Entrenament del discriminador
        with tf.GradientTape() as tape:
            predictions = self.discriminator(combined_images)
            d_loss = self.loss_fn(labels, predictions)
        grads = tape.gradient(d_loss, self.discriminator.trainable_weights)
        self.d_optimizer.apply_gradients(
            zip(grads, self.discriminator.trainable_weights)
        )

        # apartat 9: Generar nou batch de soroll + etiquetes
        random_latent_vectors = keras.random.normal(
            shape=(batch_size, self.latent_dim), seed=self.seed_generator
        )
        random_vector_labels = ops.concatenate(
            [random_latent_vectors, one_hot_labels], axis=1
        )

        # apartat 10: Etiquetes falses
        misleading_labels = ops.zeros((batch_size, 1))

        # apartat 11: Entrenament del generador
        with tf.GradientTape() as tape:
            fake_images = self.generator(random_vector_labels)
            fake_image_and_labels = ops.concatenate(
                [fake_images, image_one_hot_labels], -1
            )
            predictions = self.discriminator(fake_image_and_labels)
            g_loss = self.loss_fn(misleading_labels, predictions)
        grads = tape.gradient(g_loss, self.generator.trainable_weights)
        self.g_optimizer.apply_gradients(zip(grads, self.generator.trainable_weights))

        # apartat 12: Actualitzar mtriques de prdua
        self.gen_loss_tracker.update_state(g_loss)
        self.disc_loss_tracker.update_state(d_loss)

        # Retornar les mtriques
        return {
            "g_loss": self.gen_loss_tracker.result(),
            "d_loss": self.disc_loss_tracker.result(),

Resposta a la pregunta 5.1:

Es fa perqu passi de tenir forma (batch_size, num classes) a (batch_size, height, widht, num classes), de manera que cada mapa de classes tingui la mateixa resoluci espacial que les imatges i es pugui concatenar com a canals addicionals, fent que el discriminador rebi informaci de classe per a cadascun dels pxels.

Pregunta 5.3 [0,5 pts.]: Per qu a l'apartat 10 s'assigna l'etiqueta 0 a les imatges falses en entrenar el generador, si l'objectiu s que el discriminador les reconegui com a reals?

Al calcular la prdua del generador, volem que es minimitzi la distncia entre la predicci del discriminador i el valor "real". Aix forcem al generador a produir imatges que el discriminador classifiqui com a reals.

Pregunta 6.1 [0,5 pts.]: Qu passaria si s'utilitzs BinaryCrossentropy(from_logits=False) en lloc de from_logits=True? Com afectaria aix l'entrenament?
Resposta a la pregunta 6.1:

La diferncia est en si la funci de prdua aplica internament la sigmoid als logits abans de calcular la cross-entropy. En el cas "True", el model retorna valors sense activaci. En el cas "False", la prdua espera a rebre les probabilitats (sortides que ja han passat per la sigmoid). Per tant, si es fa servir "False", el clcul de la prdua probablement ser divergent o donar resultats erronis.

Pregunta 6.2 [0,5 pts.]: Quin efecte tindria reduir o augmentar la taxa d'aprenentatge (learning_rate) dels optimitzadors Adam durant l'entrenament? Per qu podria ser til fer servir taxes diferents per al generador i el discriminador?
Resposta a la pregunta 6.2:

Reduir el lr fa que cada pas sigui ms estable per ms lent. Augmentar-lo accelera la convergncia inicial, per pot implicar oscillacions o fins i tot que el model divergeixi si el lr s massa gran. Normalment el discriminador aprn molt ms rpid, ja que s una tasca ms fcil que la del generador, per tant, es pot donar el cas en que el discriminador aprengui molt rpid i el generador no trobi gradients tils. Assignar al generador un lr una mica ms baix (o al discriminador un ms baix) ajuda a equilibrar els ritmes d'aprenentatge i evita que un dels dos superi en gran mesura a l'altre.

Pregunta 6.3 [0,5 pts.]: Si durant l'entrenament observes que la prdua del discriminador disminueix molt rpidament i la del generador no millora, quines estratgies podries provar per millorar l'equilibri entre tots dos models?
Resposta a la pregunta 6.3:

  • canviar els lr: baixar el del discriminador o pujar el del generador.
  • modificar la freqncia d'acutlitzaci: Per exemple, entrenar el generador dues vegades per a cada pas del discriminador.
  • afegir regularitzaci: augmentar el droput, batch normalization ms agressiva, etc.
  • modificar aqruitectura del discriminador: redur la profunditat o el nombre de filtres per a que no s'imposi tan rpidament
  • Fer servir esquemes de lr adaptats (source arXiv:1704.00028)

7. Interpolació entre classes utilitzant el generador entrenat¶

En aquesta activitat analitzarem com el generador respon a entrades que representen classes intermdies (no estrictament definides com un nic dgit), cosa que ens permet explorar la continutat de l'espai latent i semntic aprs.

Es fan dos exercicis:

  • Interpolaci entre dues classes: se seleccionen dos dgits (per exemple, 1 i 9) i es generen etiquetes que van des d'una codificaci one-hot de la primera classe fins a la segona.   • Generaci controlada per classe: es genera una imatge per a cada dgit (del 0 al 9) amb soroll aleatori diferent, i s'observa si el generador ha aprs a produir exemples coherents per a cada classe concreta.

# Funci que genera imatges interpoladas entre dues classes
def interpolate_class(first_number, second_number):
    # Convertim les classes inicial i final a codificaci one-hot
    first_label = keras.utils.to_categorical([first_number], num_classes)
    second_label = keras.utils.to_categorical([second_number], num_classes)
    first_label = ops.cast(first_label, "float32")
    second_label = ops.cast(second_label, "float32")

    # Interpolem entre les dues etiquetes (lnea recta en l'espai de classes)
    percent_second_label = ops.linspace(0, 1, num_interpolation)[:, None]
    percent_second_label = ops.cast(percent_second_label, "float32")
    interpolation_labels = (
        first_label * (1 - percent_second_label) + second_label * percent_second_label
    )

    # Concatenem soroll + etiquetes interpolades i generem imatges
    noise_and_labels = ops.concatenate([interpolation_noise, interpolation_labels], 1)
    fake = trained_gen.predict(noise_and_labels)
    return fake

# Especifiquem les classes entre les que volem interpolar
start_class = 1  # Classe inicial
end_class = 9    # Classe final

# Generem les imatges falses corresponents a la interpolaci
fake_images = interpolate_class(start_class, end_class)
