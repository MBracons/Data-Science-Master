Informació de la pràctica

Les xarxes generatives adversàries (GANs) ens permeten generar dades como imatges, vídeos o àudio a partir d'una entrada aleatòria. Normalment, aquesta entrada es mostreja d'una distribució normal abans de passar per una sèrie de transformacions que la converteixen en quelcom plausible, com una imatge, un so o un clip de vídeo.

Tot i això, una DCGAN senzilla no ens permet controlar l'aparença (per exemple, la classe) de les mostres generades. Per exemple, si una GAN genera dígits manuscrits, no podríem triar quin número volem que aparegui: els resultats serien aleatoris.

Per poder controlar allò que es genera, necessitem condicionar la sortida de la GAN a una entrada semàntica, com la classe d'una imatge.

En aquesta pràctica (PAC), construirem una GAN Condicional que sigui capaç de generar dígits manuscrits condicionats a una classe concreta.

3. Càlcul del nombre de canals d'entrada per al generador i el discriminador¶

5. Implementació de la classe ConditionalGAN¶

En aquesta secció es defineix una classe personalitzada que estén keras.Model, i permet controlar manualment el procés d'entrenament d'una GAN condicional.

A diferència dels models estàndard, una GAN requereix dues xarxes entrenades de forma alternada:

  • El discriminador, que intenta distingir entre imatges reals i generades.
  • El generador, que intenta crear imatges prou realistes per “enganyar” el discriminador.

El mètode train_step està estructurat per blocs numerats (apartats) per entendre cada etapa de l'entrenament, des de la preparació de dades fins a l'actualització de pesos.

Una GAN condicional hi afegeix una capa de complexitat: s'entrena amb imatges i també amb etiquetes que indiquen la classe desitjada.

Llegeix atentament el codi i respon les preguntes següents:

class ConditionalGAN(keras.Model):
    def __init__(self, discriminator, generator, latent_dim):
        super().__init__()
        # Assignar discriminador i generador
        self.discriminator = discriminator
        self.generator = generator

        # Dimensió del vector d'entrada aleatori (soroll)
        self.latent_dim = latent_dim

        # Generador de llavors per a la reproducibilitat
        self.seed_generator = keras.random.SeedGenerator(1337)

        # Mètriques per al seguiment de les pèrdues del generador i del discriminador
        self.gen_loss_tracker = keras.metrics.Mean(name="generator_loss")
        self.disc_loss_tracker = keras.metrics.Mean(name="discriminator_loss")

    @property
    def metrics(self):
        # Llista de mètricas que es reportaran durant l'entrenament
        return [self.gen_loss_tracker, self.disc_loss_tracker]

    def compile(self, d_optimizer, g_optimizer, loss_fn):
        # Configurar optimizadors i funció de pèrdua
        super().compile()
        self.d_optimizer = d_optimizer
        self.g_optimizer = g_optimizer
        self.loss_fn = loss_fn

    def train_step(self, data):
        # Desempaquetar les dades reals (imatges i etiquetes one-hot)
        real_images, one_hot_labels = data

        # apartat 1: Expandir i repetir etiquetes
        image_one_hot_labels = one_hot_labels[:, :, None, None]
        image_one_hot_labels = ops.repeat(
            image_one_hot_labels, repeats=[image_size * image_size]
        )
        image_one_hot_labels = ops.reshape(
            image_one_hot_labels, (-1, image_size, image_size, num_classes)
        )

        # apartat 2: Generar vectors latents aleatoris i concatenar-los amb etiquetes
        batch_size = ops.shape(real_images)[0]
        random_latent_vectors = keras.random.normal(
            shape=(batch_size, self.latent_dim), seed=self.seed_generator
        )
        random_vector_labels = ops.concatenate(
            [random_latent_vectors, one_hot_labels], axis=1
        )

        # apartat 3: Generar imatges falses a partir del generador
        generated_images = self.generator(random_vector_labels)

        # apartat 4: Concatenar imatges falses amb les seves etiquetes
        fake_image_and_labels = ops.concatenate(
            [generated_images, image_one_hot_labels], -1
        )

        # apartat 5: Concatenar imatges reals amb les seves etiquetes
        real_image_and_labels = ops.concatenate([real_images, image_one_hot_labels], -1)

        # apartat 6: Unir el batch d'imatges reals i falses
        combined_images = ops.concatenate(
            [fake_image_and_labels, real_image_and_labels], axis=0
        )

        # apartat 7: Crear etiquetes: 1 per a falses, 0 per a reals
        labels = ops.concatenate(
            [ops.ones((batch_size, 1)), ops.zeros((batch_size, 1))], axis=0
        )

        # apartat 8: Entrenament del discriminador
        with tf.GradientTape() as tape:
            predictions = self.discriminator(combined_images)
            d_loss = self.loss_fn(labels, predictions)
        grads = tape.gradient(d_loss, self.discriminator.trainable_weights)
        self.d_optimizer.apply_gradients(
            zip(grads, self.discriminator.trainable_weights)
        )

        # apartat 9: Generar nou batch de soroll + etiquetes
        random_latent_vectors = keras.random.normal(
            shape=(batch_size, self.latent_dim), seed=self.seed_generator
        )
        random_vector_labels = ops.concatenate(
            [random_latent_vectors, one_hot_labels], axis=1
        )

        # apartat 10: Etiquetes falses
        misleading_labels = ops.zeros((batch_size, 1))

        # apartat 11: Entrenament del generador
        with tf.GradientTape() as tape:
            fake_images = self.generator(random_vector_labels)
            fake_image_and_labels = ops.concatenate(
                [fake_images, image_one_hot_labels], -1
            )
            predictions = self.discriminator(fake_image_and_labels)
            g_loss = self.loss_fn(misleading_labels, predictions)
        grads = tape.gradient(g_loss, self.generator.trainable_weights)
        self.g_optimizer.apply_gradients(zip(grads, self.generator.trainable_weights))

        # apartat 12: Actualitzar mètriques de pèrdua
        self.gen_loss_tracker.update_state(g_loss)
        self.disc_loss_tracker.update_state(d_loss)

        # Retornar les mètriques
        return {
            "g_loss": self.gen_loss_tracker.result(),
            "d_loss": self.disc_loss_tracker.result(),
        }
Pregunta 5.1 [0,5 pts.]: A l'apartat 1 del mètode train_step, per què cal expandir i repetir la codificació one-hot de les classes abans de concatenar-les amb les imatges?
Resposta a la pregunta 5.1:

Es fa perquè passi de tenir forma (batch_size, num classes) a (batch_size, height, widht, num classes), de manera que cada mapa de classes tingui la mateixa resolució espacial que les imatges i es pugui concatenar com a canals addicionals, fent que el discriminador rebi informació de classe per a cadascun dels píxels.

Pregunta 5.2 [0,5 pts.]: Què passaria si no tornessis a generar un nou batch de soroll a l'apartat 9 i reutilitzessis el mateix que vas fer servir per entrenar el discriminador a l'apartat 2?
Resposta a la pregunta 5.2:

Si reutilitzem el mateix soroll per als dos passos d'entrenament, el generador simplement aprendria a ajustar-se a aquells vectors concrets de soroll que el discriminador ja ha vist. Això faria que es redueixi la diversitat de mostres generades i fa que el discriminador ho tingui més fàcil per identificar-les com a sintètiques. En canvi, soroll nou, ens assegura que cada actualització del generador sigui sobre entrades fresques i promou una major varietat i estabilitat en l'entrenament.

Pregunta 5.3 [0,5 pts.]: Per què a l'apartat 10 s'assigna l'etiqueta 0 a les imatges falses en entrenar el generador, si l'objectiu és que el discriminador les reconegui com a reals?
Resposta a la pregunta 5.3:

Al calcular la pèrdua del generador, volem que es minimitzi la distància entre la predicció del discriminador i el valor "real". Així forcem al generador a produir imatges que el discriminador classifiqui com a reals.

Pregunta 6.1 [0,5 pts.]: Què passaria si s'utilitzés BinaryCrossentropy(from_logits=False) en lloc de from_logits=True? Com afectaria això l'entrenament?
Resposta a la pregunta 6.1:

La diferència està en si la funció de pèrdua aplica internament la sigmoid als logits abans de calcular la cross-entropy. En el cas "True", el model retorna valors sense activació. En el cas "False", la pèrdua espera a rebre les probabilitats (sortides que ja han passat per la sigmoid). Per tant, si es fa servir "False", el càlcul de la pèrdua probablement serà divergent o donarà resultats erronis.

Pregunta 6.2 [0,5 pts.]: Quin efecte tindria reduir o augmentar la taxa d'aprenentatge (learning_rate) dels optimitzadors Adam durant l'entrenament? Per què podria ser útil fer servir taxes diferents per al generador i el discriminador?
Resposta a la pregunta 6.2:

Reduir el lr fa que cada pas sigui més estable perè més lent. Augmentar-lo accelera la convergència inicial, però pot implicar oscil·lacions o fins i tot que el model divergeixi si el lr és massa gran. Normalment el discriminador aprèn molt més ràpid, ja que és una tasca més fàcil que la del generador, per tant, es pot donar el cas en que el discriminador aprengui molt ràpid i el generador no trobi gradients útils. Assignar al generador un lr una mica més baix (o al discriminador un més baix) ajuda a equilibrar els ritmes d'aprenentatge i evita que un dels dos superi en gran mesura a l'altre.

Pregunta 6.3 [0,5 pts.]: Si durant l'entrenament observes que la pèrdua del discriminador disminueix molt ràpidament i la del generador no millora, quines estratègies podries provar per millorar l'equilibri entre tots dos models?
Resposta a la pregunta 6.3:

  • canviar els lr: baixar el del discriminador o pujar el del generador.
  • modificar la freqüència d'acutlització: Per exemple, entrenar el generador dues vegades per a cada pas del discriminador.
  • afegir regularització: augmentar el droput, batch normalization més agressiva, etc.
  • modificar aqruitectura del discriminador: reduïr la profunditat o el nombre de filtres per a que no s'imposi tan ràpidament
  • Fer servir esquemes de lr adaptats (source arXiv:1704.00028)

# Funció que genera imatges interpoladas entre dues classes
def interpolate_class(first_number, second_number):
    # Convertim les classes inicial i final a codificació one-hot
    first_label = keras.utils.to_categorical([first_number], num_classes)
    second_label = keras.utils.to_categorical([second_number], num_classes)
    first_label = ops.cast(first_label, "float32")
    second_label = ops.cast(second_label, "float32")

    # Interpolem entre les dues etiquetes (línea recta en l'espai de classes)
    percent_second_label = ops.linspace(0, 1, num_interpolation)[:, None]
    percent_second_label = ops.cast(percent_second_label, "float32")
    interpolation_labels = (
        first_label * (1 - percent_second_label) + second_label * percent_second_label
    )

    # Concatenem soroll + etiquetes interpolades i generem imatges
    noise_and_labels = ops.concatenate([interpolation_noise, interpolation_labels], 1)
    fake = trained_gen.predict(noise_and_labels)
    return fake

# Especifiquem les classes entre les que volem interpolar
start_class = 1  # Classe inicial
end_class = 9    # Classe final

# Generem les imatges falses corresponents a la interpolació
fake_images = interpolate_class(start_class, end_class)
